# Skill: Reporting

## Propósito

Generar informes claros, concisos y accionables sobre el estado de proyectos, progreso de tareas y situación operativa.

---

## Capacidades

- Generar status reports de proyectos
- Producir resúmenes ejecutivos
- Crear resúmenes de reuniones
- Documentar revisiones semanales
- Sintetizar información de múltiples fuentes
- Estructurar información para diferentes audiencias
- Destacar riesgos e impedimentos

---

## Inputs Requeridos

- Estado actual del proyecto (`memory/active_projects.md`)
- Período a reportar
- Audiencia del informe (equipo, stakeholders, ejecutivos)
- Tipo de informe solicitado

---

## Outputs

- Status report estructurado
- Resumen ejecutivo (máximo 1 página)
- Acta de reunión
- Revisión semanal con indicadores

---

## Herramientas Compatibles

- Datos de memoria (`memory/`)
- Datos de ClickUp (via `operations_agent`)

---

## Límites

- No inventa datos — solo reporta lo que existe en memoria o ClickUp
- No hace predicciones sin base histórica
- No filtra información negativa — reporta el estado real

---

## Principios de un Buen Reporte

1. **Relevante**: Solo incluye lo que importa para la audiencia
2. **Accionable**: Cada observación tiene una acción asociada
3. **Honesto**: Incluye bloqueos y riesgos, no solo logros
4. **Conciso**: El resumen ejecutivo cabe en 5 minutos de lectura
5. **Consistente**: Usa el mismo formato cada vez para facilitar comparación
